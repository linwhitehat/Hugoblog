从Hexo.Next到Hugo.MemE

 2020-07-23  2020-07-27  1355  3 分钟  

去年10月左右开始搭建了基于 Next主题的 Hexo 博客，同时也对博客做了很多定制化内容，近期准备升级新版本主题和框架，但是却发现出现了一些问题：

Next缺乏长期稳定的维护团队，目前经历三代维护人员的变更，都是因为前一代维护作者不再维护而带来的变更，而这带来一些奇怪的错误无法在社区解决的情况，例如我遇到的这个问题，目前在新旧两代维护社区中都未得到原因和准确处理办法。


  • ：Next的定制化程度很高，但是维护团队在大版本或功能更新等情况下会改变主题的结构，官方团队也建议在跨大版本升级主题的时候做好旧版本备份，这将加大对博客主题做了大量个性化内容修改的更新成本，而且每次都需要准确的对照主题结构变化进行相应修改，花费精力不亚于一次博客迁移。
  • ：尽管 Hexo 框架下博客生成和加载速度有所提升，但是并不算最优，个性化配置下的内容显得有点杂乱和冗长，对主题的原版修改可能在出现问题时不便于版本回退。

最终在接近使用10个月的情况下，我还是准备放弃 Hexo，由于旧版本博客以 Github Page 形式发布，因此会维持原先的开源博客， 但是维护和修改的情况会大幅度减少，迁移后的 Hugo 博客将在新域名上搭建，预计在今年10月份之前完成。

迁移到 Hugo 框架下，我选择了友链中一个博主开发的 MemE 主题，简洁的主题风格以及对 Next 的样式继承，对于从 Next 迁移过来十分友好，其功能也十分强大[1]：

  1. 完全的深色模式支持
  2. 支持 InstantClick 极速点击
  3. 完全的多语言支持
  4. 可自定义的 SVG 图标
  5. 支持 HTML/CSS/JS 的压缩和 JS 的捆包
  6. 代码高亮、脚注、Todo 列表、定义列表、Emoji
  7. 字体排版
    • 自定义字体、字号、行间距
    • 自定义文章内容宽度
    • 段首缩排、两端对齐
    • 首字下沉、小型大写
    • 图片说明、图片脚注
    • 着重号
    • 中文标点符号字形纠正
  8. 归档页面、分类页面、标签云页面
  9. Atom╱RSS 订阅
  10. 分节标题锚点
  11. 图片外链、视频外链
  12. 自定义页脚
  13. 404 页面、置顶╱排序文章
  14. PWA 和 Service Worker 的更新通知
  15. Google Analytics、Google Site Verification、Google AdSense
  16. 不蒜子页面浏览量和站点访客数、站点访问量
  17. Disqus 评论、Valine 评论、Utterances 评论
  18. KaTeX 和 MathJax 3.0 公式支持（包括化学方程式）
  19. SEO 和社交网络相关：Sitemap、JSON-LD、Open Graph、Twitter Cards
  20. 支持部署在星际文件系统（IPFS）
  21. ······

基于 Go 开发的 Hugo 框架在生成页面的速度确实令人惊喜，同时 Meme 主题采取与 Hugo 框架根目录结构相同的这点，对于主题的变化几乎可以在 Hugo 结构下进行修改而不需要修改主题内容，对于博客主题个性化十分友好，同时很多在 Next 下需要自定义的功能也被集成到了 MemE 中，基于种种情况，我毫不犹豫迁移了博客，最终迁移及摸清楚大致的主题个性化花费了一天左右的时间，整体而言，目前博客的呈现效果和 Hexo 的时候差异不会太大，在个性化部分基本与之前对 Next 的修改差不多，明显的一些变化有：

  1. 首页从完全自制的首页以及 Next 的文章摘要页变为单独的诗词页面，保持了我自制首页的简洁观念，同时视觉效果更好一点；
  2. 侧边栏被删去，但是我希望在阅读博客时侧边存在一个固定目录，这部分可能需要自己加上；
  3. 顶栏减少了导航键，部分导航键功能准备添加到首页作为诗词下方的控件，同时背景变为朦胧的彩色。

尽管迁移了博客，但我依旧保留了旧博客的存在，文章不会做重定向的迁移，但是会向新博客做一个导引，希望在新博客下能创作更好的文章，更踏实的发表优质内容。
