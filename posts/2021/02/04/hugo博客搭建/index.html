之前的博客是基于Hexo.Mist搭建完成的，由于个人对前端的学习有限，我了解到Hugo.Meme后，更喜欢这种简约感，因此将博客转至此。最初的博客雏形一般都比较朴素，功能也有限，通过优化主题来进行美化博客外观和拓展博客功能依旧是转移博客后的必要工作。旧博客仍然可以继续访问，Hexo和Hugo搭建的博客均以GithubPage的形式发布。

HUGO博客

去年12月我将博客内容迁移到Hugo，选择了主题Meme，机缘来自我的友链中荷戟独彷徨和reuixiy。起初搭建个人博客是希望锻炼一下自己的前端动手能力和记录一下生活，选择了简易的Hexo.Next，同时也查阅了很多资料进行博客个性化修改。之所以选择迁移博客框架，主要还是想在一个简洁画风下更多的分享内容，将博客的内容专注在文字上。
这篇博客将完整讲述博客迁移的过程，对遇到的一些问题及解决方法也会一起阐述。

HUGO安装

这部分内容基本遵循官方文档进行。

  1. 下载与安装
    在Hugo的开源地址下载所需发布版本，我的电脑系统是Windows-64，因此选择下图中的对应版本下载。下载后的压缩包解压之后便是HUGO的安装目录，将此目录加入系统环境变量。之后在CMD命令行下输入hugo version，显示正确的版本信息等即表示成功安装。
  2. 创建网站
    要搭建自己的博客，需要先建立本地网站目录。在CMD命令行下进入作为离线存储博客内容的目录，输入以下命令新建网站目录。
1

hugo new site Myblog

博客迁移

  1. 博客主题
    搭建完HUGO环境之后，在官方的主题网站选择喜欢的主题框架并下载到本地。在刚刚新建的网站目录Myblog下新建一个目录themes，在这个目录下存放下载的主题文件。启用博客主题需要修改HUGO配置文件，即在博客目录Myblog下的config.toml。在配置文件中添加theme内容，具体如下：
baseURL = "https://linwhitehat.github.io"
title = "林深时见璐"
languageCode = "zh-CN"
hasCJKLanguage = true
theme = "meme"
  1. 内容迁移
    启用博客主题之后，将旧博客中的内容迁移到新博客中，这需要了解HUGO的博客基本框架，具体如下：
.
├── archetypes  # 内容类型，在创建新内容时自动生成内容的配置
├── content     # 网站内容，Markdown 文件
├── data
├── layouts     # 网站模版，选择主题后会将主题中的 layouts 文件夹中的内容复制到此文件夹中
├── static      # 包含 CSS、JavaScript、Fonts、media、image 等，决定网站的外观。
├── themes      # 存放主题文件
├── public      # 可公开静态内容
└── config.toml # 网站的配置文件

博客文章是存储在content目录下，图片、前端渲染等是存储在static目录下，因此我们将原先Hexo博客下的内容分别迁移到相应目录下即可。

博客本地预览

在正式发布博客之前，可以在本地预览博客内容并进行调试修改。在CMD下执行下述命令进行预览：

# 启动 HUGO 
hugo server -D
# 启动成功会显示下述内容
Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)
Press Ctrl+C to stop

此时，在本地浏览器访问http://localhost:1313/便能看到博客。

GithubPage发布

将本地博客最终发布到GithubPage前，需要生成可发布的静态文件。在CMD下执行下述命令：

# 生成博客网站
hugo

此时在博客目录Myblog下的public目录会生成可用于发布的博客内容，将整个目录上传到GithubPage仓库即可。

避雷

上述步骤完成，基本博客内容的迁移就完成了，至于HUGO主题框架的具体配置可参照配置文件进行基础修改。但是要正确部署还有不少雷区需要注意，如果没有正确认识这些问题可能博客并无法像本地预览的效果一样。

  1. baseURL设置
    一般而言，搭建GithubPage时我们只会把二级域名当作博客地址，但是由于我制作了博客主页，因此增加了子目录的网站地址才是我博客内容的真正地址。此时，容易误解baseURL而将博客网站的地址填写在配置中，因为Hexo配置的时候便需要标明博客的具体网站地址（包括子目录），然而实际却非如此。正确配置举例对比如下：
# Hexo URL
https://linwhitehat.github.io/Hugoblog/
# Hugo baseURL
https://linwhitehat.github.io/

如果配置错误可以在部署之后在网站控制台找到错误信息，一般是显示资源找不到的错误，对比错误资源地址可以确认正确baseURL的填写。
2. 完整性检验
对于资源完整性检验在搭建GithubPage博客中不是很友好，因此需要将其设置为false才能正确链接和渲染前端资源文件，否则会显示Failed to find a valid digest in the 'integrity' attribute for resource等错误。

个人博客主页

很多现有的博客框架主题都将文章作为博客首页，也有少数有自身主页的博客主题，但是修改起来可能需要更多熟悉主题框架结构。为了更便利和自主搭建自己的博客首页，我自定义了一个静态网站作为自己的博客首页，修改方便而且不容易和别人的撞衫。目前主页源码已经开源，欢迎读者作为自己的博客模板进一步加工制作属于自己的博客首页。

结束

博客包含的可能不只是文字，图片和简单的前端知识，我想也有审美，艺术和丰富的建站体验。在2021年，我会更专注于所学，静下心，将内容输出为系列文章，提高博客的质量。那么就以这篇为开始吧！

彩蛋

微软资源告别龟速下载
知名的MSDN告诉了我们微软的资源可以便捷的找到，但是镜像资源的庞大让没有迅雷会员的我总是苦于漫长的等待。当我想建立一个微软操作系统的虚拟机进行某个实验时，如果长时间去等待下载，一来让我有种一件事还没完成但是却不能马上完成的焦躁，其次也让我耗费时间去记录此时的想法以等待具备可实验的环境时进行实操。那么，怎么解决呢？告别MSDN+迅雷的组合，使用微软官方+IDM，尽情享受高速体验下载微软资源的快乐吧！

  1. 微软官方镜像链接
    访问微软官网，此时看到的官网显示的是立即下载工具而没有镜像下载链接。接着，打开浏览器的开发者工具，按照下图标记的位置，打开Remote devices模式即手机模式，之后刷新网站并按照提示选择便能看到镜像下载链接。

  2. IDM下载
    在浏览器中安装IDM插件，点击上一步中的镜像下载后跳转到IDM进行下载即可。
